# Proof file parser and types for forge verify

## Outcome

`VerifyOptions` and `VerifyResult` types exist in `src/types.ts`. A proof parser function reads `.md` proof files from a given directory, skips `index.md` and `readme.md`, and returns each proof split into two parts: automated check items (from the "Automated Tests" section) and human-only steps (from "Manual Verification" and "Visual Checks" sections).

## Acceptance Criteria

- `VerifyOptions` interface added to `src/types.ts` with fields: `proofDir`, `outputDir?`, `prompt?`, `cwd?`, `model?`, `maxTurns?`, `maxBudgetUsd?`, `quiet?`, `dryRun?`
- `VerifyResult` type added to `src/types.ts` capturing per-proof outcome: `proofPath`, `proofName`, `status` (`'pass' | 'fail' | 'skipped'`), `automatedPassed`, `automatedTotal`, `humanSteps: string[]`, `costUsd?`, `durationSeconds?`
- `ForgeResult.type` union in `src/types.ts` extended to include `'verify'` (currently `'run' | 'audit' | 'review' | 'define' | 'prove'`)
- Parser collects all `.md` files from the specified directory, skipping `index.md` and `readme.md` (case-insensitive), and throws a clear error if no files are found
- Parser extracts lines under the `## Automated Tests` heading into a list of check items
- Parser extracts lines under `## Manual Verification` and `## Visual Checks` headings into human-only steps
- Parsed structure is returned as a plain object usable directly by the check runner without further filesystem access
- TypeScript compiles without errors
- Existing tests still pass

## Context

- `src/types.ts` — add `VerifyOptions` and `VerifyResult` alongside existing `ProveOptions`; follow the same interface shape
- `src/prove.ts` — the proof files being parsed are generated by `runProve()`; proof sections use `##` headings: "Automated Tests", "Manual Verification", "Visual Checks", "Edge Cases"
- `src/audit.ts` — reference for directory collection logic that skips `index.md` / `readme.md`
- **Naming conflict**: `src/verify.ts` already exists for build/test verification (`detectVerification`, `runVerification`, monorepo scoping); the new verify command implementation goes in a separate file — `src/proof-runner.ts`
- Parser can live directly in `src/proof-runner.ts` rather than a separate module
